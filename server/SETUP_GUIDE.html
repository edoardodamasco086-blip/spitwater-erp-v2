📖 Developer Guide · Phase 1 & 2

Setting up the
ERP backend on Windows 11

This guide takes you from a blank Windows 11 machine to a fully running Express API connected to your SQL Server — step by step, no experience assumed.

[ Your Browser / Postman ] | | HTTP requests to /api/* v [ Node.js + Express ] ← server.js (port 3000) | | SQL queries via mssql driver v [ SQL Server 2014 — SERVER006 ] | | tables, triggers, stored procs v [ Development_04052026 database ] ← your schema
💡
React comes later. In Phase 1 & 2 we only build the backend. The React frontend (Vite) will be added in Phase 3 and will connect to this same API. For now you test with Postman or a browser.

Project folder structure

Where every file lives and why

C:\Projects\spitwater-erp\ │ ├── server\ ← Express backend (what we're building now) │ ├── server.js ← app entry point — runs this to start │ ├── package.json ← lists all npm packages │ ├── .env ← YOUR secrets (DB password, JWT key) — NEVER commit │ ├── .env.example ← safe template — commit this │ ├── .gitignore ← tells git to ignore .env and node_modules │ │ │ ├── config\ │ │ ├── db.js ← SQL Server connection pool (reads .env) │ │ └── logger.js ← Winston logger (writes to /logs) │ │ │ ├── middleware\ │ │ ├── auth.js ← verifies JWT on protected routes │ │ └── errorHandler.js ← catches all errors, formats response │ │ │ ├── routes\ │ │ ├── auth.js ← /api/auth/* — login, logout, refresh │ │ └── users.js ← /api/users/* — user management │ │ │ ├── utils\ │ │ ├── jwt.js ← sign & verify JWT tokens │ │ └── password.js ← bcrypt hash & verify │ │ │ ├── scripts\ │ │ ├── test-db.js ← run to check DB connection │ │ └── seed-admin.js ← creates first org + super admin │ │ │ ├── logs\ ← auto-created, app.log + error.log │ └── node_modules\ ← auto-created by npm install, never touch │ ├── client\ ← React app (Phase 3 — not built yet) │ └── ERP_SCHEMA_COMPLETE.sql ← already deployed to SERVER006

The schema is already done. You already ran ERP_SCHEMA_COMPLETE.sql in SSMS — the 148 tables are live. This guide connects Node.js to those tables.
1

Install Node.js on Windows 11

Node.js is the runtime that executes the JavaScript server code.

a
Download Node.js LTS
Get the Long Term Support version — stable and recommended

Go to nodejs.org → click the big green "LTS" button → download the .msi installer.

📌
Get version 20 or 22 LTS. Avoid odd-numbered versions (those are development/unstable). Our package.json requires ≥ 18.
b
Run the installer
Accept all defaults — including the "Tools for Native Modules" checkbox

Double-click the downloaded .msi file. Click Next through everything. When you see the checkbox for "Automatically install the necessary tools" — tick it. This installs Python and Build Tools which some npm packages need.

⚠️
This opens a black PowerShell window at the end that installs extra tools. Let it finish — it can take 5–10 minutes. Don't close it.
c
Verify the install
Open a new terminal and check the versions

Press Win + R → type cmd → Enter. Then run:

Command Prompt
C:\> node --version
v22.x.x   ← should show v18 or higher

C:\> npm --version
10.x.x    ← should show 9 or higher
✅
If you see version numbers you're good. If you get "not recognized" — close the terminal completely and open a new one, then try again.

3

Install npm packages

npm reads package.json and downloads all dependencies into node_modules\

a
Open terminal in the server folder

In File Explorer, navigate to C:\Projects\spitwater-erp\server\ then click the address bar at the top, type cmd and press Enter. This opens a terminal already in that folder.

💡
Alternative: Right-click on the server folder → "Open in Terminal" (Windows 11 has this built in).
b
Run npm install
cmd — inside server\
C:\Projects\spitwater-erp\server> npm install

This downloads ~10 packages. Takes 30–60 seconds. You'll see a lot of text scrolling. When done you should see something like added 237 packages.

⚠️
If you see red errors about node-gyp or missing Python — run npm install --ignore-scripts instead. The mssql driver doesn't require native compilation so this is safe.
c
Install nodemon globally (for auto-restart)
cmd
C:\Projects\spitwater-erp\server> npm install -g nodemon

Nodemon watches your files and restarts the server automatically every time you save a .js file. Used with npm run dev.

4

Create your .env config file

This is where all your database credentials and secrets live

🔐
The .env file is secret. It contains your database password and JWT signing key. It is listed in .gitignore so git will never commit it. Never paste its contents into Slack, email, or chat.
a
Copy the template

In the server\ folder, make a copy of .env.example and name it exactly .env (dot env, no other extension).

cmd
C:\Projects\spitwater-erp\server> copy .env.example .env
b
Edit the .env file
Open with Notepad, VS Code, or any text editor
.env — fill in YOUR values
# Server
PORT=3000
NODE_ENV=development

# Database — change SERVER006 to your machine name or IP
# If SQL Server is on THIS machine: use localhost
# If it's on another machine on LAN: use the machine name or IP
DB_SERVER=SERVER006
DB_PORT=1433
DB_DATABASE=Development_04052026
DB_USER=sa
DB_PASSWORD=YourActualSaPasswordHere
DB_ENCRYPT=false
DB_TRUST_CERT=true
DB_WINDOWS_AUTH=false

# JWT — generate a real secret with the command below
JWT_SECRET=CHANGE_THIS_TO_SOMETHING_LONG_AND_RANDOM
JWT_EXPIRES_IN=8h
JWT_REFRESH_EXPIRES_IN=7d

# React dev server (leave as-is for now)
CORS_ORIGIN=http://localhost:5173

BCRYPT_ROUNDS=10
LOG_LEVEL=debug
c
Generate a proper JWT secret

The JWT_SECRET must be a long random string. Run this command to generate one:

cmd
C:\> node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
a3f8c2d1e9b7...  ← copy this whole string into JWT_SECRET in your .env
⚠️
Never use a short or guessable JWT secret in any environment. Anyone who knows the secret can forge authentication tokens and log in as any user.
d
How to find your DB_SERVER value

You need the hostname or IP of the machine running SQL Server 2014.

Options
# Option A: SQL Server on this same Windows 11 machine
DB_SERVER=localhost

# Option B: SQL Server on another machine by name (SERVER006)
DB_SERVER=SERVER006

# Option C: SQL Server on another machine by IP
DB_SERVER=192.168.1.50

# Option D: Named SQL Server instance (e.g. SERVER006\SQLEXPRESS)
DB_SERVER=SERVER006
DB_INSTANCE=SQLEXPRESS   ← add this line

To find SERVER006's IP: on SERVER006, open cmd and type ipconfig. Look for IPv4 Address.

5

Enable SQL Server TCP/IP (one-time setup)

SQL Server blocks network connections by default — you need to turn this on

⚠️
Do this on the machine running SQL Server 2014 (SERVER006), not necessarily your dev machine. If they're the same machine, do it there.
1
Open SQL Server Configuration Manager

Press Win + R → type SQLServerManager12.msc → Enter.
If that doesn't work search Start Menu for "SQL Server Configuration Manager".

2
Enable TCP/IP

In the left panel: SQL Server Network Configuration → Protocols for MSSQLSERVER. In the right panel, right-click TCP/IP → Enable.

3
Set TCP port to 1433

Right-click TCP/IP → Properties → click the IP Addresses tab. Scroll to the very bottom to IPAll. Set TCP Port = 1433. Clear the TCP Dynamic Ports field (leave it blank).

4
Restart SQL Server service

In Configuration Manager left panel: SQL Server Services → right-click SQL Server (MSSQLSERVER) → Restart.

5
Open Windows Firewall port 1433

Only needed if Node.js is on a different machine to SQL Server. Run this in an Administrator cmd:

cmd (Administrator)
C:\> netsh advfirewall firewall add rule name="SQL Server 1433" protocol=TCP dir=in localport=1433 action=allow

.env settings explained

Every variable and what it does

.env — full reference
# ── Server ───────────────────────────────────────────────────────────
PORT=3000                 The port Express listens on. 3000 is the convention.
NODE_ENV=development     "development" = verbose errors. "production" = safe errors.

# ── Database ─────────────────────────────────────────────────────────
DB_SERVER=SERVER006      Machine name or IP of the SQL Server host.
DB_PORT=1433             SQL Server port. Default is 1433.
DB_DATABASE=Development_04052026    Database name. Must match SSMS.
DB_USER=sa               SQL Auth username. "sa" is the system admin.
DB_PASSWORD=...          SQL Auth password. Keep it safe.
DB_ENCRYPT=false         true only if SQL Server has a proper SSL cert.
DB_TRUST_CERT=true       Bypasses SSL cert validation. Fine for LAN dev.
DB_WINDOWS_AUTH=false    true = use Windows login, ignore DB_USER/PASSWORD.
DB_INSTANCE=SQLEXPRESS   Only needed for named instances (e.g. SERVER\SQLEXPRESS).

# ── JWT ──────────────────────────────────────────────────────────────
JWT_SECRET=...           64+ char random string. Signs all tokens. KEEP SECRET.
JWT_EXPIRES_IN=8h       Access token lifetime. 8h = user stays logged in 8 hours.
JWT_REFRESH_EXPIRES_IN=7d    Refresh token lifetime. 7d = silent re-auth for 7 days.

# ── CORS ─────────────────────────────────────────────────────────────
CORS_ORIGIN=http://localhost:5173    Vite dev server. Add your prod domain in prod.

# ── Bcrypt ───────────────────────────────────────────────────────────
BCRYPT_ROUNDS=10         Password hashing cost. 10=dev (fast), 12=prod (secure).

# ── Logging ──────────────────────────────────────────────────────────
LOG_LEVEL=debug          debug=everything, info=normal, warn=warnings, error=only errors.

Common errors & fixes

❌
ConnectionError: Failed to connect to SERVER006:1433
🔧
Fix checklist:
1. SQL Server service is running (check Services or SQL Config Manager)
2. TCP/IP is enabled in SQL Config Manager (Step 5)
3. Port 1433 is open in Windows Firewall
4. DB_SERVER in .env matches the actual machine name or IP
5. Named instance? Add DB_INSTANCE=SQLEXPRESS to .env
❌
Login failed for user 'sa'
🔧
Fix: In SSMS, right-click the server → Properties → Security → make sure "SQL Server and Windows Authentication mode" is selected. Then right-click the sa login → Properties → make sure it's enabled and the password matches your .env.
❌
Cannot find module 'mssql' or similar
🔧
Fix: You haven't run npm install yet, or you're in the wrong folder. Make sure your terminal is inside server\ and run npm install.
❌
401 Unauthorized on protected routes
🔧
Fix: You need to include the Authorization header. In Postman: Headers tab → add Authorization: Bearer YOUR_TOKEN_HERE. Get the token from the login endpoint first.
❌
nodemon is not recognized
🔧
Fix: Run npm install -g nodemon. If still not found, close the terminal and open a new one, or use node server.js instead of npm run dev.
❌
EADDRINUSE: port 3000 already in use
🔧
Fix: Another process is using port 3000. Either kill it: in cmd run netstat -ano | findstr :3000 then taskkill /PID <number> /F. Or change PORT=3001 in your .env.
